Lecture 39 HIGH PRIORITY

IP Routing — III: Autonomous Systems & Path Vector Routing

Covers the architecture of Autonomous Systems, IGP vs EGP protocols, BGP path vector routing mechanics, loop prevention, policy routing, and a recap of OSPF LSA types.

1. Autonomous Systems (AS)

The internet is not managed by a single administrative authority. Instead, it is divided into Autonomous Systems — logical portions of a large IP network, each under one administrative control (an organisation, ISP, university, etc.). Each AS is assigned a globally unique AS number by an international regulatory authority.

Definition: An Autonomous System is a logically grouped set of routers and networks operated under a single administrative domain, identifiable by a unique AS number, capable of connecting to other ASes (public or private, same or different organisations).

Border Routers

Each AS has a designated Border Router (AS Boundary Router) that maintains summarised information about the entire AS, updated by IGPs (OSPF/RIP) internally. These border routers exchange summarised information with border routers of other ASes, enabling inter-domain path computation.

2. Routing Protocols in AS
Interior Gateway Protocols (IGPs)

Exchange routing information within an AS. The routing policy is entirely at the discretion of the owning organisation. Examples: OSPF (link-state) and RIP (distance-vector). Each AS can independently choose which IGP to run.

Exterior Gateway Protocols (EGPs)

Exchange summary routing information between ASes. Border routers advertise reachability of their AS's networks to neighbouring border routers. The canonical example is BGP (Border Gateway Protocol).

3. Areas within an AS

A single AS can be further divided into Areas — each a collection of routers, networks, and hosts within the AS, identified by an Area ID. All networks within an area must be connected (area = connected subgraph).

Backbone Area (Area 0): A special area inside every AS. All other areas must connect to the backbone. It acts as the primary area; all others are secondary. Routers within area 0 are called backbone routers.

Router Roles in an AS

Router Type Role Scope
Internal Router Floods routing info within its area Single area
Area Border Router (ABR) Summarises area info; connects area to backbone (area 0) Between areas within AS
Backbone Router Operates in Area 0; maintains backbone topology Area 0
AS Boundary Router (ASBR) Lives in Area 0; holds summarised AS info; communicates with other ASes via EGP (BGP) Between ASes
A backbone router can also be an Area Border Router. These roles are not mutually exclusive. The ASBR is the outward-facing representative of the entire AS to the rest of the internet.

Information Flow (Hierarchy)

Other ASes inter-domain via BGP

Backbone Routers (Area 0) backbone topology via OSPF

Internal Routers (Area N) flood info within area via OSPF/RIP
4. Path Vector Routing

Path vector routing is the mechanism used for inter-domain (EGP) routing. Unlike distance-vector (stores cost to destination) or link-state (stores full topology), path vector stores the actual sequence of ASes a packet must traverse.

Routing Table Format

Field Description
Destination Destination network address
Next Router Immediate next-hop router
Path Ordered list of AS IDs to traverse (e.g. AS14 → AS23 → AS67)
Example entry: To reach network N1, next router = R1, path = AS14, AS23, AS67. Each AS boundary router prepends its own AS ID when advertising to neighbours, growing the path list hop by hop.

Key Properties of Path Vector Routing

Path Vector Messages

AS boundary routers advertise reachability of networks within their AS to neighbouring boundary routers (two boundary routers on the same network = neighbours). These advertisements contain the path list. Boundary routers are first updated internally by OSPF/RIP before advertising outward.

Each receiving router verifies the advertised path against its local policy before accepting it.

Loop Prevention

Because each path advertisement is an ordered list of AS IDs, a loop is trivially detectable — if the receiving router's own AS ID appears anywhere in the path, that message is discarded. A repeated AS ID in the path = loop detected.

Example of a looped path: AS1 → AS2 → AS6 → AS9 → AS6 → AS13 — AS6 appears twice. This message is dropped.

Policy Routing

This is the biggest advantage of path vector over other protocols. An AS can define policies such as: "never route through AS X," "must route through AS Y," "prefer path through friendly ASes." If a received path violates local policy, the message is discarded — a clean, enforceable mechanism unavailable in pure distance-vector or link-state schemes.

Path Attributes

Well-Known Attributes

Must be recognised by every BGP router. Further divided into Mandatory (must be present and propagated, e.g. AS_PATH, ORIGIN, NEXT_HOP) and Discretionary (recognised but not required in every update).

Optional Attributes

Not required to be recognised. Can be transitive (pass on even if not understood) or non-transitive (discard if not understood). Allow extensibility without breaking older routers.

5. Border Gateway Protocol (BGP) — RFC 1771

BGP is the internet's inter-domain routing protocol, based on path vector. It is the protocol that makes the global internet's AS-level routing work.

Why Not Distance Vector or Link State for Inter-AS?

Protocol Type Problem at Internet Scale
Distance Vector Assumes all routers share a common distance metric. Different ASes have different priorities and meanings for metrics — a stable, loop-free topology becomes impossible. Also gives no information about which ASes will be traversed.
Link State Flooding is completely impractical at internet scale — consuming enormous bandwidth, and convergence of a full global topology is computationally and practically infeasible. Every router would need the entire internet's topology.
Path Vector (BGP) No distance metric needed. Carries ordered AS path list. Loop detection is trivial. Policy enforcement is natural. Scales to internet size.

BGP vs Distance Vector — Key Difference

Path vector does not compute a cost or distance estimate. It lists all ASes to be visited. This is both more informative (you know the exact route) and more policy-flexible than a simple cost number.

BGP Messages

All BGP messages are sent over a TCP connection on port 179. There are four message types:

Type 1
OPEN

Initiates a BGP session with a neighbouring router. Establishes the connection.

Type 2
UPDATE

Advertises new reachable paths or withdraws previously advertised paths. Carries path attributes.

Type 3
KEEPALIVE

Periodic heartbeat — signals that the router is alive and ready to receive messages. Maintains the TCP session.

Type 4
NOTIFICATION

Sent when an error or exceptional condition is detected. Closes the BGP session after delivery.

BGP Functional Procedures

Procedure Messages Used Purpose
Neighbour Acquisition OPEN + KEEPALIVE Establish BGP session with a neighbouring AS boundary router
Neighbour Reachability Periodic KEEPALIVE Verify that neighbour is still operational
Network Reachability UPDATE (broadcast) Advertise which networks are reachable via this AS, and the preferred path
Note: The BGP RFC (1771) does not specify how a router learns the address of its BGP neighbours — that is left to the network administrator to configure manually.

Path Advertisement Propagation

When AS2 receives an advertisement from AS1 saying "to reach N1, path = AS1", AS2 prepends itself and advertises to AS3: "to reach N1, next router = R2, path = AS2 → AS1." AS3 in turn advertises: "path = AS3 → AS2 → AS1." Each hop adds itself to the front of the path, building the full ordered list.

6. OSPF Recap (RFC 1247) — IGP / Intra-AS

OSPF operates within an AS. It uses a value in the 8-bit Protocol field of the IP header. Metric is flexible: minimum delay (RTT), maximum throughput, reliability, etc. It divides the AS into areas and uses link-state advertisements (LSAs) to share topology.

OSPF Link State Advertisement (LSA) Types

LSA Type Originator Describes
Router Link Every router All links/interfaces of the originating router within its area
Network Link Designated Router (DR) All routers attached to the transient (multi-access) network
Summary Link to Network Area Border Router (ABR) Routes to networks in other areas — flooded by ABR into the area
Summary Link to AS Boundary Area Border Router (ABR) Routes to the AS Boundary Router — used so routers know how to reach external destinations
External Link AS Boundary Router (ASBR) Routes to networks external to the AS — flooded throughout the AS

OSPF Packet Types

Type 1
Hello

Neighbour discovery and keepalive within OSPF.

Type 2
Database Description

Summary of link-state database exchanged during initial synchronisation.

Type 3
Link State Request

Request for specific LSA entries not yet received.

Type 4
Link State Update

Carries the actual LSAs (router link, network link, summary link to network/ASBR, external link).

Type 5
Link State Acknowledgement

Acknowledges received LSAs — ensures reliable flooding.

The Link State Update packet is the most important OSPF packet type for exam purposes. The LSA types it carries directly map to the AS area hierarchy: router links for internal topology, summary links for cross-area info, and external links for inter-AS info.
Concept Summary

The Big Picture

The internet is carved into ASes, each with its own internal routing (IGP: OSPF/RIP). Between ASes, BGP (an EGP based on path vector routing) governs how reachability is advertised. Within each AS, a strict hierarchy exists: internal routers → area border routers → backbone routers → AS boundary router → BGP to the outside world.

Why Path Vector Wins for Inter-AS Routing

Distance vector fails because ASes don't share a common metric and gives no AS traversal info. Link-state flooding is impossible at global scale. Path vector carries just enough: an ordered list of ASes to cross, no metric computation, with built-in loop detection (repeated AS in path = drop) and natural policy enforcement (forbidden AS in path = drop).

BGP in One Sentence

BGP is a path-vector EGP that runs over TCP (port 179), where AS boundary routers exchange OPEN/UPDATE/KEEPALIVE/NOTIFICATION messages to build and maintain routing tables containing destination networks, next-hop routers, and the ordered list of ASes to traverse — with loop prevention and policy filtering built in.
