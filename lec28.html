Lecture 28 HIGH PRIORITY

IP Addressing (IPv4) II — CIDR

Subnetting worked examples, the all-zero/all-one subnet problem, supernetting, hierarchical address allocation, and Variable Length Subnet Mask (VLSM).

▶ watch lecture
Quick Recap

CIDR Addressing Format

An IP address in CIDR is written as a.b.c.d/n where n is the number of leading bits that form the network prefix. The remaining 32 − n bits are the host address within that network.

Example: 10111111.10110100.01010011.11101011 with netmask 11111111.11110000.00000000.00000000 → first 12 bits = subnet address, remaining 20 bits = host address.

The subnet mask is ANDed with the IP address to extract the network portion. The host portion identifies the specific device within that subnet.

Core Concept

Dividing a Network into Subnets

Given a network block 203.110.0.0/16: the first 16 bits identify the network, and the remaining 16 bits are free for host addressing. To create subnets, you borrow bits from the host portion and use them as a subnet identifier. The more bits you borrow, the more subnets you can make — but the fewer hosts each subnet can hold.

Visual hint: The lecture works through this example step-by-step on a diagram at 01:37 and 02:12. Worth watching if the binary arithmetic feels unclear.

Why 3 bits to create 3 subnets — not 2?

With 2 borrowed bits you get four possible patterns: 00, 01, 10, 11. But 00 (all-zeros) and 11 (all-ones) are reserved and cannot be used as subnet identifiers (explained below). That leaves only 01 and 10 — just two usable subnets. To get three usable subnets you need 3 bits, giving patterns 000–111; using 100, 101, 110 avoids both all-zeros and all-ones.

2 bits (insufficient)
patterns00, 01, 10, 11
reserved00 (subnet zero) & 11 (all-one)
usable01, 10 → only 2 subnets
3 bits (correct)
patterns000 – 111
reserved000 & 111
usable001, 010, 011, 100, 101, 110 → 6 subnets

Result for 203.110.0.0/16 → 3 subnets

Borrow 3 bits → new prefix = 16 + 3 = /19. Remaining 13 bits for host addressing. Using subnet IDs 100, 101, 110:

Subnet Subnet ID (binary) Network Address Prefix Host bits
Subnet 1 100 203.110.128.0 /19 13 bits (8190 hosts)
Subnet 2 101 203.110.160.0 /19 13 bits (8190 hosts)
Subnet 3 110 203.110.192.0 /19 13 bits (8190 hosts)
Important Rule

The All-Zero and All-One Subnet Problem

These two subnet identifiers cause address ambiguity and must be avoided.

Subnet Zero (All 0s)
Consider 192.168.0.0/16 split with 1 borrowed bit, using 0 as the subnet identifier. The resulting subnet's network address = 192.168.0.0 — identical to the original network's address. A router cannot distinguish between the two, causing routing ambiguity.
All-One Subnet
Using 1 as the subnet identifier for 192.168.0.0/16 gives subnet 192.168.128.0/17. Its broadcast address (set all host bits to 1) = 192.168.255.255 — identical to the original network's broadcast. This causes broadcast storms and routing confusion.
Summary: All-zero subnet → subnet network address collides with parent network address. All-one subnet → subnet broadcast address collides with parent network broadcast address. Both are illegal by convention.
Worked Example

CIDR + Supernetting — IIT Kharagpur Scenario

The Computer and Informatics Centre (CIC) of IIT KGP receives the address block 203.110.0.0/19 from PNIC (India's IP allocation authority). This gives 13 host bits → up to 213 − 2 = 8190 usable addresses.

Three departments need subnets:

DepartmentHosts RequiredHost bits neededCalculation
CSE 2000 11 bits 211 = 2048 ≥ 2000 ✓
VGSOM 500 9 bits 29 − 2 = 510 ≥ 500 ✓
EE 500 9 bits 29 − 2 = 510 ≥ 500 ✓
The problem: The largest subnet (CSE) needs 11 host bits. Out of the 13 available bits, that leaves only 2 bits for subnet identification — which, as established above, is insufficient to safely create 3 subnets. A fix is needed.

Solution: Apply Supernetting (Combine VGSOM + EE)

VGSOM and EE each have 500 hosts. Combined: 1000 hosts → requires only 10 host bits (210 = 1024). Now the problem collapses to two subnets at the top level (CSE vs. VGSOM+EE), which requires only 2 subnet bits — and with 3 total bits available (13 − 10 = 3), this works.

Level 1 Split — 203.110.0.0/19 → two subnets

Borrow 2 bits from host space. New prefix = 19 + 2 = /21. Assign subnet IDs 10 (CSE) and 01 (VGSOM+EE):

SubnetSubnet IDNetwork AddressPrefixRemaining host bits
CSE 10 203.110.16.0 /21 11 bits
VGSOM + EE (supernet) 01 203.110.8.0 /21 11 bits

Level 2 Split — 203.110.8.0/21 → VGSOM and EE

Inside the VGSOM+EE supernet block, borrow 2 more bits. New prefix = 21 + 2 = /23. Assign IDs 10 (VGSOM) and 01 (EE):

SubnetSubnet IDNetwork AddressPrefixRemaining host bits
VGSOM 10 203.110.12.0 /23 9 bits (510 hosts)
EE 01 203.110.10.0 /23 9 bits (510 hosts)
Final allocation summary: CSE → 203.110.16.0/21 | VGSOM → 203.110.12.0/23 | EE → 203.110.10.0/23. This is hierarchical addressing — two levels of subnetting, routed by an intermediate router placed between the supernet boundary and the individual department subnets.

Advanced Concept

Fixed-Length vs Variable-Length Subnet Mask (VLSM)

In the IIT KGP example above, the same prefix length was used for all subnets at each level (both Level 1 subnets used /21; both Level 2 subnets used /23). This is fixed-length subnet masking (FLSM).

CIDR does not require this. It natively supports Variable Length Subnet Mask (VLSM) — different subnets under the same parent network can have different prefix lengths. This allows address space to be allocated more efficiently based on actual host requirements.

Fixed-Length (FLSM)
All subnets at a given level share the same prefix length. Simpler to manage but wastes addresses — small subnets get the same allocation as large ones.
Variable-Length (VLSM)
Each subnet gets a prefix length matched to its actual host count. More efficient use of address space. Requires CIDR-aware routing (classless routing protocols like OSPF, EIGRP, BGP).

VLSM Worked Example — 202.110.0.0/20

Given 202.110.0.0/20: 12 host bits available → 212 − 2 = 4094 usable addresses. Required: S1 = 1000 hosts, S2 = 500 hosts, S3 = 500 hosts.

SubnetHostsHost bits neededPrefixSubnet bits from /20
S1 1000 10 bits /22 2 bits borrowed
S2 500 9 bits /23 3 bits borrowed
S3 500 9 bits /23 3 bits borrowed

S1 gets a /22 mask, while S2 and S3 each get a /23 mask — these are different lengths under the same parent, which would be illegal in FLSM but is perfectly valid in VLSM/CIDR.

Critical constraint — prefix uniqueness: When assigning subnet identifiers under VLSM, no subnet's identifier bits must be a binary prefix of another subnet's identifier. For example, if S1 uses subnet ID 10 (for /22), then S2 cannot use 101 (for /23) because 10 is a prefix of 101 — a packet destined for a host in S1 could falsely match S2's entry. Choose IDs such that no one is a prefix of another.

Concept Summary

Key Takeaways

Subnetting
Borrowing bits from host space
To create N subnets, borrow ⌈log₂(N+2)⌉ bits from the host portion (the +2 accounts for avoided all-zero and all-one IDs). New prefix = original prefix + borrowed bits. Each subnet gets fewer host bits as a result.
Forbidden
All-zero and all-one subnet IDs
All-zero subnet: its network address equals the parent network address. All-one subnet: its broadcast address equals the parent broadcast address. Both cause ambiguity in routing and must be avoided.
Supernetting
Combining smaller subnets into a larger block
When there aren't enough subnet bits to split into the required number of independent subnets, smaller departments can be logically combined under a single intermediate router. The combined block is then re-split at the next level. This is hierarchical IP allocation.
VLSM
Variable Length Subnet Mask
CIDR allows different subnets under the same parent to carry different prefix lengths, matched to actual host requirements. More efficient than FLSM. Requires that subnet identifiers are prefix-free — no subnet ID can be a binary prefix of another.
