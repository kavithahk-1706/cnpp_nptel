Lecture 48 HIGH PRIORITY

Data Link Layer — Ethernet

Shared access networks, MAC protocols (Aloha → CSMA/CD), Ethernet frame structure, collision detection, and exponential backoff. Core exam material — understand every mechanism here.

Shared Access Networks

Multiple nodes share a single physical link (bus, ring, wireless). Any transmission by one node is received by all other nodes — no intermediate switches. This means all devices are in the same broadcast domain by default.

Because the medium is shared, simultaneous transmissions cause collisions. MAC (Medium Access Control) protocols govern how nodes take turns, with two primary goals:

Fairness
Every node gets equitable access to the channel — no node is starved of bandwidth.
Performance
Maximize channel utilization. Protocols must be lightweight enough to run on resource-constrained devices.
Multiple Access Methods — Taxonomy
Fixed Assignment
Channel is partitioned — each node gets a fixed slice of bandwidth (TDMA, FDMA, CDMA). Essentially circuit switching. Inefficient: idle nodes waste their allocated slot. Used in wireless/cellular environments.
Contention-Based
Nodes compete equally for bandwidth and handle collisions when they occur. No pre-allocation. Examples: Aloha, Ethernet (CSMA/CD). This is what we focus on.
Token / Reservation
Nodes take turns; only the node holding the token may transmit. Orderly but requires coordination overhead. Example: Token Ring.
Aloha Protocol Family

Developed by Norm Abramson at the University of Hawaii (1970) for packet radio networks. The first random-access MAC protocol. Foundation for everything Ethernet is built on.

Pure (Unslotted) Aloha

Simplest approach: when ready, transmit immediately. No synchronization, no coordination. If no ACK is received within a timeout period, assume collision → wait a random delay → retransmit.

Problem: A frame sent at time t₀ can collide with any frame sent in the window [t₀ − 1, t₀ + 1] — double the vulnerable period. Maximum channel utilization: ~18%. Terrible.

Delay tradeoff: too short → many collisions. Too long → channel underutilized.

Slotted Aloha

Time is divided into equal-sized slots (each = one packet transmission time). Nodes may only begin transmitting at the start of a slot. On collision, retransmit in a future slot with probability p until successful.

Slotting halves the vulnerable period → maximum utilization improves to ~36%. Still low, but a meaningful improvement over pure Aloha.

Neither version checks whether the channel is busy before transmitting — that's the key limitation Ethernet's carrier sense solves.

Ethernet — Background

Developed by Bob Metcalfe and colleagues at Xerox PARC in the mid-1970s. Roots directly in the Aloha packet radio network. Standardized jointly by Xerox, DEC, and Intel in 1978. The first practical Local Area Network (LAN).

IEEE 802.3 defines both MAC and physical layer details. Ethernet has scaled through generations:

Standard Speed Notes
IEEE 802.3 2 → 10 Mbps Original CSMA/CD Ethernet
IEEE 802.3u 100 Mbps Fast Ethernet
IEEE 802.3z 1000 Mbps (1 Gbps) Gigabit Ethernet

Ethernet is the dominant LAN technology worldwide — cheap, fast, and standards-based.

Ethernet Technologies — Physical Variants

The naming convention for Ethernet standards follows: [speed][signaling][cabling].

10Base2
Thin Coax Bus
10 Mbps, ≤185 m cable length. Bus topology — all nodes tap into a single coaxial backbone. Repeaters used to connect multiple segments (Layer 1 device: amplifies signal, boosts SNR). Packets travel in both directions.
10BaseT / 100BaseT
Twisted Pair + Hub
T = Twisted Pair. Enables star topology via hubs. Max run ≈ 89–100 m. Hub = multiport repeater (Layer 1). All ports share the same collision and broadcast domain — effective bandwidth degrades with more ports.
Hub vs Switch: A hub is just a multiport repeater — Layer 1. All connected nodes share one collision domain and one broadcast domain. Effective bandwidth = total bandwidth ÷ number of ports (worst case). A switch (Layer 2) solves the collision problem by giving each port its own collision domain.

Carrier Sense — What Aloha Was Missing

Ethernet's key improvement over Aloha: listen before you transmit. Before sending, a node checks whether the medium is currently in use (carrier sensed). If busy, wait. If idle, transmit.

Aloha didn't do this because it was a wireless protocol — in wireless, the contention problem exists at the receiver, not the sender. Two issues unique to wireless make carrier sense difficult: hidden terminal (node C can't hear node A, so C doesn't know A is transmitting to B) and exposed terminal (node B hears A's ongoing transmission and incorrectly withholds its own unrelated transmission). Ethernet runs on wired media where these don't apply.

CSMA/CD — Ethernet's MAC Protocol

Carrier Sense Multiple Access / Collision Detection — the algorithm Ethernet actually uses.

CS — Carrier Sense
Send only if medium is idle
MA — Multiple Access
Many nodes share the channel
CD — Collision Detection
Detect via voltage difference; stop immediately

MAC Algorithm — Step by Step

If line is idle: Send packet immediately. Max frame size = 1500 bytes. Must wait 9.6 µs between back-to-back frames.
If line is busy (carrier sensed): Wait until idle, then transmit immediately. This strategy is called 1-persistent sending — as soon as it's free, fire.
If collision detected: Stop sending immediately. Broadcast a 48-bit jam signal so all nodes on the network know a collision occurred. Then apply exponential backoff and retry.

CSMA/CD State Diagram

The full decision loop (refer slide time 26:38 for the diagram):

Packet arrives→Sense Carrier
Carrier free→Send packet
No collision detected→Transmission complete ✓
Collision detected→Jam channel → CalcBackoff() → wait(b) → attempts++
attempts < 16→Retry: sense carrier again
attempts == 16→Discard packet → report error to host

Collision is detected by monitoring voltage on the wire — simultaneous transmissions produce an abnormal combined voltage that the adapter recognises as a collision.

Collision Detection — The 2T Constraint

Consider nodes A and B at opposite ends of the wire. A starts transmitting at time 0. A's signal reaches B at time T. B, seeing the line idle, also starts transmitting — collision occurs at B. B's collision signal propagates back and reaches A at time 2T. A must still be transmitting at time 2T to detect the collision — this sets the minimum frame size.

IEEE 802.3 specifies max 2T = 51.2 µs. This corresponds to a max host separation of ~2500 m (effectively ~1500 m accounting for cable propagation). At 10 Mbps, 1 bit takes 0.1 µs to send, so 512 bits = 64 bytes takes 51.2 µs → minimum Ethernet frame size = 64 bytes.

MAC Address Structure

Format: XX:XX:XX:XX:XX:XX — 6 bytes = 48 bits. Each manufacturer is assigned a unique address range; every NIC worldwide has a globally unique MAC (barring cloning). Special cases:

Unicast → first bit = 0 (specific destination) Broadcast → FF:FF:FF:FF:FF:FF (all 1s) Multicast → first bit = 1
Exponential Backoff

After a collision, nodes don't retry immediately — that would just cause another collision. Instead they use binary exponential backoff to spread retransmission times apart.

AttemptChoose K fromDelay
1st{0, 1}K × 51.2 µs
2nd{0, 1, 2, 3}K × 51.2 µs
nth{0, 1, ..., 2ⁿ − 1}K × 51.2 µs
Max K1023cap on backoff growth
16th attempt—Give up; report error to host
Why random? If delay were fixed/deterministic, colliding nodes would retransmit simultaneously again — lockstep failure. Randomness breaks the symmetry.
Why not just use a small fixed K? Works fine for few hosts. With many nodes, many simultaneous collisions need a wide spread of K values to avoid piling up again. The exponentially growing range accommodates network load growth.
Switched Ethernet

Replacing hubs with Layer 2 switches fundamentally improves Ethernet performance. Switches forward and filter frames based on MAC addresses, maintaining a forwarding table.

Hub (Repeater)
LayerL1 — physical only
CollisionSingle shared domain — all ports collide

Switch (L2)
LayerL2 — reads MAC addresses
CollisionEach port = separate collision domain

Switches only copy frames to the port where the destination MAC is located — unlike repeaters which blindly blast signal everywhere. This is the critical difference: selective forwarding vs dumb amplification.

Concept Summary

Aloha → CSMA/CD Evolution

Pure Aloha (1970): Send whenever ready. No channel check. 18% max utilization.
Slotted Aloha: Send only at slot boundaries. 36% max utilization.
CSMA (Carrier Sense): Check if medium is idle before sending. Major improvement — avoids collisions with ongoing transmissions.
CSMA/CD: CSMA + detect collisions mid-transmission and abort. Send jam signal. Use exponential backoff. This is Ethernet.

Type Field = Demultiplexer

The 16-bit Type field in an Ethernet frame tells the receiving host which network-layer protocol should process the payload (e.g. 0x0800 = IPv4, 0x0806 = ARP). It's how Layer 2 hands off to Layer 3 correctly — the demultiplexing key.
