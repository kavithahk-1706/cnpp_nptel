Data Link Layer — Basic Concepts

MAC sublayer structure, frame format, MAC techniques (CSMA/CD & Token Passing), LAN addressing, and MAC protocol taxonomy.

Data Link Layer Structure

The Data Link Layer (Layer 2) is divided into two sublayers, standardised under the IEEE 802 family of standards which covers both Layer 2 and the Physical layer.

LLC — Logical Link Control

Handles upper-layer negotiation and connectivity. Exposes a LLC Service Access Point (LSAP) to upper-layer protocols, identifying which process or service should receive the data.

MAC — Medium Access Control

Interfaces directly with the physical layer. Manages access to the shared medium and carries the hardware (MAC) address that uniquely identifies a network interface card worldwide.

Cross-layer optimisation: Although layered design enables interoperability, strict layer separation can limit performance. Cross-layer optimisation — jointly tuning multiple layers — is an active research area, especially for QoS, security, and multimedia applications.
Generic MAC Frame Format

When the MAC sublayer receives a LLC PDU (Protocol Data Unit) from LLC, it wraps it into a MAC frame by adding a header and a trailer.

Field Description
MAC Control Control information for MAC protocol operation, e.g. priority level.
Destination MAC Address Hardware address of the destination's physical attachment point (NIC) on the LAN. This is a hop-to-hop address — the next stop (e.g. a switch or wireless access point), not the final endpoint.
Source MAC Address Hardware address of the sender's physical attachment point (NIC).
LLC PDU (Payload) Data received from the LLC sublayer (which contains the network-layer datagram wrapped in LLC header).
CRC (Trailer) Cyclic Redundancy Check — appended at the end. The receiving station recomputes the CRC and compares it to detect transmission errors.
MAC address (hardware address): Burned into the NIC at manufacture. Globally unique. Layer 2 operates purely in terms of MAC addresses — it has no knowledge of IP. ARP (Address Resolution Protocol) bridges this gap by resolving an IP address to the corresponding MAC address at each hop.
MAC Techniques

MAC techniques govern how stations get the right to transmit over a shared medium. There are two broad categories:

Synchronous

A fixed capacity is dedicated to each connection — analogous to circuit-switched FDM or TDM. Not suitable for LANs/MANs because station demand is unpredictable; pre-allocated bandwidth is wasted when a station is idle.

Asynchronous

Capacity is allocated dynamically in response to actual demand. The dominant approach for LANs. Subdivided into three strategies: Round Robin, Reservation, and Contention.

Asynchronous Sub-categories

Strategy Mechanism Best for Weakness
Round Robin Each station is granted the right to transmit in turn. After finishing, it passes the right to the next station in the logical sequence. When many stations have data to send uniformly over time. Wasteful if some stations have nothing to transmit — they still consume their turn.
Reservation Time is divided into slots (like synchronous TDM). A station reserves slots in advance for an extended period before transmitting. Steady streaming traffic — voice, bulk file transfer. Over-allocates for bursty traffic; requires advance coordination.
Contention No centralised control — stations compete freely for the medium. No turn-taking, no pre-reservation. Bursty, sporadic traffic — interactive terminal/host traffic. Simple to implement. Efficient under light load; degrades under heavy load due to collisions.
Medium Access Control Methods

Two principal MAC-level access methods exist in practice:

CSMA/CD

Carrier Sense Multiple Access with Collision Detection. Used in bus topologies. The dominant technique — forms the basis of traditional Ethernet.

Token Passing (Control Token)

A permission token circulates around a logical ring. Applicable to both bus and ring topologies. Less common today; mostly superseded by Ethernet.

CSMA/CD — Deep Dive

CSMA/CD is used only in bus-type networks where all nodes share a single common communication medium. It combines three mechanisms:

Carrier Sense (CS)

Before transmitting, a station listens to the medium. If busy, it waits until the channel becomes idle. If idle, it transmits.

Multiple Access (MA)

Multiple stations share the same medium. Even after sensing idle, two stations may begin transmitting simultaneously, causing a collision.

Collision Detection (CD)

While transmitting, the station keeps listening. A collision produces a voltage fluctuation detectable by the hardware. On detection, a jam signal is broadcast to alert all stations.

CSMA/CD Operation — Step by Step

1. Source station assembles a MAC frame (destination address + data + control info).
2. Listen to the medium — carrier sensing. If busy, wait. If idle, transmit.
3. Continue listening while transmitting — collision detection.
4. If a collision is detected (voltage fluctuation), immediately transmit a random bit pattern — the jam sequence — to notify all stations.
5. All stations involved in the collision back off for a random time period, then retry from step 2.
6. If the back-off retry limit is exceeded, the transmission is aborted (link failure declared).

Collision Detection Time

The worst-case time to detect a collision is 2 × end-to-end propagation delay. This is because station A starts transmitting; just before A's signal reaches station B at the far end of the bus, B also starts transmitting. B detects the collision immediately, but A must wait for B's jam signal to propagate all the way back — a full round-trip propagation delay — before it realises a collision occurred.

Control Token / Token Passing

An alternative MAC access method where a special permission token circulates among all stations in a logical ring. Only the station holding the token may transmit.

Key Properties

Works on both bus and ring physical topologies by establishing a logical ring overlaid on the physical medium. A single token is created at one node and passed from node to node in the logical ring sequence.

Token Passing Operation — Step by Step

1. Station waits for the token to arrive.
2. Station captures the token (gains exclusive transmission right).
3. If it has data to send: transmit frames. If not: immediately release the token.
4. A station may hold the token only for a fixed allocated time limit. Once the time expires, it must release the token regardless of remaining data.
5. Token is released to the next node in the logical ring.
Status: Token passing (Token Ring — IEEE 802.4/802.5) was historically used but is now largely obsolete. Modern networks are almost entirely Ethernet (CSMA/CD) based.
LAN Addressing

Communication involves three agents: processes (software running on a machine), stations (end devices), and networks (interconnecting infrastructure). To deliver data to the correct process on the correct machine, the data link layer uses two levels of addressing:

MAC Address (Hardware Address)

Identifies a physical interface (NIC) — the station's attachment point to the LAN. One-to-one relationship between station and MAC address. Wired or wireless, every NIC has a globally unique MAC address burned in by the manufacturer.

LLC Address (LSAP)

Identifies an LLC user — a particular process, service, or hardware port within a station. The LLC Service Access Point (LSAP) is how the data link layer knows which upper-layer protocol or process should receive the incoming data.

Encapsulation Flow at Layer 2

When an IP datagram arrives from the network layer:

User data (IP datagram)
↓ LLC adds LLC header (containing source/dest LSAP)
LLC PDU
↓ MAC adds MAC header (source/dest MAC addresses) + MAC trailer (CRC)
MAC Frame — transmitted on the physical medium to the next hop

The MAC header's destination address is always the next-hop MAC address, not the final destination. At every hop, the MAC frame is stripped, the LLC PDU is extracted, and a new MAC frame is constructed for the next segment.

MAC Protocols — Taxonomy

All MAC protocols fall into three broad classes:

Class Principle Example
Channel Partitioning Divide the channel into smaller pieces (time slots or frequency bands). Allocate a piece to each node for exclusive use. TDMA, FDMA
Random Access Allow collisions to occur freely; define mechanisms to recover from them. CSMA/CD (Ethernet)
Shared / Controlled Access Tightly coordinate access to the shared medium so collisions are avoided entirely. Token Passing

Design Goals of Any MAC Protocol

Efficient — maximise channel utilisation Fair — no node gets unfair preference Simple — low implementation complexity Decentralised — no single point of control
Concept Summary
Data Link Layer = LLC (upper-layer interface) + MAC (medium access + hardware addressing). IEEE 802 standardises both.

MAC Frame = MAC Control | Dest MAC | Src MAC | LLC PDU | CRC. All addresses are hop-to-hop, not end-to-end.

MAC Techniques: Synchronous (fixed allocation, like circuit switching) vs Asynchronous (dynamic — Round Robin / Reservation / Contention).

CSMA/CD: Sense before transmit → detect collision by voltage fluctuation → jam signal → random backoff → retry. Worst-case collision detection = 2 × propagation delay. Foundation of Ethernet.

Token Passing: Permission token circulates in a logical ring. Holder may transmit within a time limit. Collision-free by design. Now largely obsolete.

LAN Addressing: Two levels — MAC address (identifies NIC/physical attachment) and LLC address/LSAP (identifies process or upper-layer service). ARP maps IP → MAC at each hop.

MAC Protocol Goals: Efficient, Fair, Simple, Decentralised.
