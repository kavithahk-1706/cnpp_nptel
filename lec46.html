Lecture 46 med priority

Data Link Layer — Overview

Introductory lecture for a series on the Data Link Layer (L2). Covers its position in the protocol stack, its two sub-layers (LLC and MAC), the core problems it solves, IEEE 802 standards, and how framing/encapsulation works.

1 — Where DLL Sits

The Data Link Layer is Layer 2 in both the OSI and TCP/IP stacks, sandwiched between the Network Layer (L3) above and the Physical Layer (L1) below. In many references — and especially in the IEEE 802 standard family — the Data Link and Physical layers are treated together as a combined layer.

Application SMTP, HTTP, FTP…
Transport TCP / UDP
Network IP — logical, end-to-end
Data Link ← you are here Hop-to-hop, MAC frame
Physical Bits on the wire
Even when two L3 routers need to communicate, they must still come down to Layer 2 to resolve the next-hop MAC address (via ARP) and push the frame onto the physical medium. DLL is therefore unavoidable — even at the inter-router level.

Recall the domain-isolation properties by layer:

Layer 1 — Hub
CollisionOne shared domain — all devices collide
BroadcastOne shared broadcast domain
EfficiencyLowest
Layer 2 — Switch
CollisionEach port = isolated collision domain
BroadcastStill one shared broadcast domain
EfficiencyBetter — no collisions across ports

A router (L3) splits both collision and broadcast domains — broadcasts cannot cross a router interface.

2 — The Two Sub-layers

The Data Link Layer is split into two distinct sub-layers, each with a specific responsibility:

LLC — Logical Link Control
Interfaces with the layer above (Network Layer). Provides Service Access Points (SAPs) so different upper-layer protocols can share the same data link. Handles flow and error control at the link level. The LLC is relatively stable — it doesn't change when the physical medium changes.
MAC — Medium Access Control
Interfaces with the layer below (Physical Layer). Assembles data into frames with addressing and error-detection fields; governs when and how a device accesses the shared transmission medium. Multiple MAC options can exist for the same LLC (e.g., Ethernet vs. Wi-Fi vs. Token Ring all using the same LLC).
The LLC/MAC split gives flexibility: changing the physical medium (Ethernet → Wi-Fi) requires changing only the MAC protocol, not the LLC or any higher layers.
3 — Core Problems at Layer 2
Issue 1
Sharing a Wire
When multiple hosts share a single medium (bus topology), there must be a mechanism to coordinate who transmits and when. A direct dedicated wire per pair becomes unscalable; a shared bus or switch fabric is needed.
Issue 2
Collision — Simultaneous Transmission
When two devices transmit at the same time on a shared medium, their signals collide and corrupt each other. Both frames must be retransmitted, wasting bandwidth. The principle: listen before you speak. If multiple transmit simultaneously anyway, both must back off and retransmit — this is the basis of CSMA/CD.
Issue 3
Addressing — Who Receives the Frame?
On a shared medium, every device sees every frame. There must be a Layer 2 addressing scheme (MAC address) so a device can identify whether a frame is meant for it. Additionally, a sender knowing only a destination IP address must resolve the corresponding MAC address — this is ARP's job.
4 — IEEE 802 Standard Family

The IEEE 802 standards govern the Data Link and Physical layers. They ensure interoperability between devices from different manufacturers. The scope spans from the Physical Layer up through the MAC and LLC sub-layers.

Standard Name Notes
802.2 Logical Link Control (LLC) Common LLC layer across all 802 MACs
802.3 CSMA/CD — Ethernet Dominant standard today; 10 Mbps → 100 Mbps → Gbps
802.4 Token Bus Largely obsolete
802.5 Token Ring Largely obsolete
802.11 Wireless LAN (Wi-Fi) Uses CSMA/CA (collision avoidance, not detection)
CSMA/CD vs CSMA/CA: Wired Ethernet detects collisions after they happen (CD = Collision Detection). Wireless cannot do this reliably, so 802.11 tries to avoid collisions before transmission (CA = Collision Avoidance). More on this in later lectures.
5 — LLC in Detail

The LLC layer transmits Protocol Data Units (PDUs) between two stations on a shared LAN, without needing intermediate switching nodes. Its payload is whatever the Network Layer handed down (IP datagram + all headers above).

Two defining characteristics of LLC:

Must support multi-access, shared-medium nature of the link Offloads medium-access details to the MAC sub-layer

LLC Service Types

Unacknowledged Connectionless
Datagram-style. No flow or error control. No delivery guarantee. Best-effort — analogous to IP at the network layer.
Connection Mode
Logical connection established before data transfer. Flow and error control maintained. Connection-oriented — analogous to TCP.
Acknowledged Connectionless
Hybrid. No prior connection setup, but each datagram is individually acknowledged. Provides reliability without connection overhead.
6 — MAC in Detail

MAC Functions

At Sender
Assembles data into frames with destination MAC address, source MAC address, and error-detection fields (e.g., CRC). Governs when to transmit on the shared medium.
At Receiver
Disassembles frames. Performs address recognition (is this frame for me?). Performs error detection. Requests retransmission if errors found.

MAC Control: Where and How

Where — Centralized
A single controller has authority to grant network access. Determines order of transmission for all stations.
Where — Distributed
Stations collectively and dynamically determine the transmission order via a consensus protocol (e.g., CSMA/CD backoff).

How access is controlled is constrained by the network topology and involves trade-offs between cost, performance, and complexity.

The NIC (Network Interface Card / LAN card) in your laptop is what physically implements the MAC sub-layer and interfaces with the Physical Layer — whether wired Ethernet, Wi-Fi, Bluetooth, etc.
7 — Encapsulation at Layer 2

Each layer wraps the payload from the layer above with its own header (and sometimes a trailer). At the Data Link Layer, this produces a MAC Frame:

Application Data
payload
TCP Header + App Data
TCP segment
IP Header + TCP segment
IP datagram
LLC Header + IP datagram
LLC PDU
MAC Header + LLC PDU + MAC Trailer
MAC Frame (L2 frame)
The MAC trailer typically contains a CRC (Cyclic Redundancy Check) for error detection — not covered in depth here, but critical. The MAC header contains source and destination MAC addresses for the current hop only. These change at every hop; the IP header (with end-to-end source/destination) remains unchanged throughout.
Concept Summary
What DLL actually does: Hop-to-hop delivery. Encapsulates IP datagrams into MAC frames, resolves next-hop MAC address, governs shared medium access, detects (and requests retransmission for) transmission errors. Every single hop along a packet's path involves a full DLL operation.
Sub-layer
LLC

Connects to Network Layer above. Provides SAPs. Handles end-to-end link services (flow, error control). Stable regardless of physical medium.

Sub-layer
MAC

Connects to Physical Layer below. Frames data, adds MAC addresses, detects errors, governs channel access. Changes with physical medium.

Dominant Standard
IEEE 802.3 (Ethernet)

CSMA/CD. Covers MAC + Physical. Speeds from 10 Mbps to multi-Gbps. 802.11 is its wireless counterpart.

Cross-layer Protocol
ARP

Lives between L2 and L3. Resolves IP address → MAC address so the data link layer knows the next-hop hardware address.
