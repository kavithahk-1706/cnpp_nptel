IP Addressing (IPv4) I — Classful Addressing

Prof. Sandip Chakraborty · IIT Kharagpur · Network Layer II

▶ Watch on NPTEL
Overview

IPv4 uses a 32-bit address to uniquely identify a host on the internet. The address is hierarchically split into a network part (identifies which network the host belongs to) and a host part (identifies the specific host within that network) — analogous to a postal address. This lecture covers the original classful addressing scheme, its limitations, and the introductory concepts behind the modern replacement: CIDR.

Classful addressing is not used today, but it is the conceptual foundation for all IP addressing. Understand it before CIDR makes complete sense.
Dotted Decimal Notation
08:13

The 32-bit IPv4 address is divided into four 8-bit chunks (octets), each represented as a decimal integer separated by dots.

Each octet spans 0–255 (since 28 − 1 = 255). Example: 203.110.30.42

To convert: read each 8-bit group as a binary integer, express in decimal, join with dots. The maximum value per octet is all-1s = 255; the minimum is all-0s = 0.

Classful Addressing — The Five Classes
04:18

The entire 32-bit address space is divided into five classes identified by the leading bits (prefix). The prefix encoding is a prefix-free code — no class identifier is a prefix of another, so a simple bit-scan unambiguously identifies the class.

Class Leading Bits Network Bits Host Bits Address Range Purpose
A 0 8 (7 usable) 24 1.0.0.0 – 127.255.255.255 Unicast — large networks
B 10 16 (14 usable) 16 128.0.0.0 – 191.255.255.255 Unicast — medium networks
C 110 24 (21 usable) 8 192.0.0.0 – 223.255.255.255 Unicast — small networks
D 1110 — (no network/host split) 224.0.0.0 – 239.255.255.255 Multicast
E 1111 — (reserved) 240.0.0.0 – 255.255.255.255 Reserved for future use
Class identification trick: Scan bits left-to-right. First 0 → Class A. First 10 → Class B. First 110 → Class C. First 1110 → Class D. First 1111 → Class E. Implemented in hardware as a bit-shift + logical AND.

Usable Network Bits

Some leading bits within the network field are consumed by the class identifier itself, leaving fewer bits for actual network numbering:

Class A
8 − 1 = 7 network bits

Leading bit "0" is fixed. 7 bits remain → 128 possible networks.

Class B
16 − 2 = 14 network bits

Leading bits "10" are fixed. 14 bits remain → 16,384 possible networks.

Class C
24 − 3 = 21 network bits

Leading bits "110" are fixed. 21 bits remain → 2,097,152 possible networks.

Class D
No network/host split

Entire address identifies a multicast group.

Multicast (Class D)
06:13

Multicast allows a single packet to be delivered to a group of hosts simultaneously, identified by one multicast group address. This differs from unicast (one-to-one) and broadcast (one-to-all-on-network).

Class D multicast range: 224.0.0.0 → 239.255.255.255
Leading bits: 1110 | First octet: 224 (11100000) to 239 (11101111)

Example analogy: sending a physical letter addressed to "all B.Tech CSE students, IIT Kharagpur" — a single address, but every member of the group receives it.

Reserved Addresses — Network & Broadcast
14:12

Within each classful network, two host addresses are reserved and cannot be assigned to actual hosts:

Network Address
Host bits All 0s
Purpose Identifies the network itself (used in routing tables)
Class A ex. 126.0.0.0
Class B ex. 189.233.0.0
Broadcast Address
Host bits All 1s
Purpose Delivers packet to every host in the network
Class A ex. 126.255.255.255
Class B ex. 189.233.255.255
All-0s and all-1s host addresses are always reserved. Never assign them to a host.
Valid Host Count Formula
17:41

Because all-0s (network address) and all-1s (broadcast address) are reserved from the host space:

Valid hosts per network = 2n − 2, where n = number of host bits.
Class Host Bits (n) Formula Max Valid Hosts
A 24 224 − 2 ≈ 16,777,214
B 16 216 − 2 65,534
C 8 28 − 2 254
The Problem with Classful Addressing
21:52

Classful addressing suffers from severe address space waste due to rigid class boundaries. Consider a network needing 255 hosts:

Class C — Not Sufficient
Max hosts 28 − 2 = 254
Needed 255
Result Cannot accommodate — one host too many
Class B — Wasteful
Max hosts 216 − 2 = 65,534
Used 255
Result ~65,279 addresses wasted
Classful addressing forces you to jump between drastically different network sizes with no middle ground. This was the primary driver for adopting CIDR.
CIDR — Classless Inter-Domain Routing
23:09

CIDR replaces fixed class boundaries with a variable-length network prefix. The core idea: break large networks into smaller pieces (subnetting) or aggregate small networks into a larger one (supernetting), allowing precise allocation of exactly the address space needed.

Subnetting
Divide a large network into multiple smaller subnets. Bits are borrowed from the host field and reassigned as a subnet number, shrinking the host space per subnet.
Supernetting
Combine multiple smaller networks (e.g., several Class C blocks) into a single larger logical network. Achieves sizes impossible under classful rules.

Address Structure Under CIDR

Under classful addressing the address was: [Network Prefix | Host Number]. Under CIDR/subnetting it becomes:

Network Prefix + Subnet Number + Host Number
← Subnet IP (used in routing) → ← Host field →

The subnet number borrows bits from the original host field. The combined network prefix + subnet number forms the subnet IP — the identifier of the sub-network the host lives in.

Subnet Mask
26:16

Without fixed class boundaries, you need additional information to know where the network part ends and the host part begins. This is the role of the subnet mask (also called netmask in Linux).

A subnet mask is a 32-bit value: a contiguous block of 1s (network bits) followed by a contiguous block of 0s (host bits). The count of leading 1s tells you how many bits belong to the network/subnet prefix.

Example — given IP address and subnet mask:

IP Address 10111111 . 10110100 . 01010011 . 11101011
Subnet Mask 11111111 . 11110000 . 00000000 . 00000000
Result First 12 bits = network/subnet address | Last 20 bits = host

Classful equivalents expressed as subnet masks:

ClassDefault Subnet MaskPrefix Length
A255.0.0.0/8
B255.255.0.0/16
C255.255.255.0/24
CIDR Notation
27:54

CIDR notation compactly expresses an IP address and its prefix length together:

Format: <IP address> / <prefix length>
Example: 191.180.83.235/12
→ First 12 bits = network address | Remaining 20 bits = host address
→ Subnet mask = 255.240.0.0 (twelve 1s, twenty 0s)

Another common example — a typical home/office network configuration:

IP Address 192.168.1.50
Subnet Mask 255.255.255.0  (/24)
Network Address 192.168.1.0
Host Identifier 50 (last 8 bits)
OS terminology: Windows uses "Subnet Mask" (dotted decimal form, e.g. 255.255.255.0). Linux uses "Netmask" or the /prefix notation. Both mean the same thing.
Exam Practice
MCQ 1: A network has 500 hosts. Which is the most appropriate classful address class?
A) Class C   B) Class B   C) Class A   D) Class D

Answer: B — Class B. Class C supports only 28−2 = 254 hosts (insufficient). Class B supports up to 65,534 hosts. Class A is overkill. Class D is multicast only.
MCQ 2: What is the broadcast address for the network 172.16.0.0/16?

Answer: 172.16.255.255 — Set all host bits (last 16 bits) to 1.
MCQ 3: IP address 200.10.5.100/28 — how many valid hosts can this subnet support?

Answer: 24 − 2 = 14 hosts. Prefix /28 → 32−28 = 4 host bits.
MCQ 4: Which class does IP address 192.168.1.1 belong to?

Answer: Class C. First octet 192 = 11000000 in binary — leading bits are 110, which identifies Class C.
Concept Summary
CIDR / Classless Variable prefix; subnetting + supernetting
Subnet Mask 32-bit mask of 1s then 0s; marks network/host boundary
Classful Addressing Fixed A/B/C/D/E classes; identified by leading bits
Dotted Decimal Four 8-bit octets (0–255) separated by dots
IPv4 Address 32 bits = Network part + Host part

Key Takeaways

IPv4 uses 32-bit addresses written in dotted decimal. Under classful addressing, the class is determined by the leading bits; Classes A, B, C each split the 32 bits into fixed-size network and host fields. The network address (all-0s host) identifies the network and the broadcast address (all-1s host) reaches every host — both are reserved. Valid hosts per network = 2n − 2.

Classful addressing wastes enormous address space when network size requirements fall between class boundaries. CIDR solves this via subnetting (splitting large networks) and supernetting (merging small ones), using a variable-length subnet mask to indicate the network prefix boundary. CIDR notation writes this as IP/prefix-length.
