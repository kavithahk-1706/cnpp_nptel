Protocol Stacks — OSI & TCP/IP + Network Devices

▶ watch lecture
Lecture Overview

This lecture covers two primary topics: (1) a formal comparison of the OSI and TCP/IP models — the number of layers each defines, how they map to one another, and where they differ — and (2) network devices, specifically what layer each device operates at and what that implies about its capabilities. There is also a brief history of the internet, which is low-priority for examination purposes but provides useful contextual grounding.

What Is a Protocol — Formal Definition ↗ ~01:11
A protocol is a controlled, precisely specified set of messages along with the rules governing when and how those messages are exchanged between two or more systems to accomplish a defined communication task. It specifies: (1) what messages exist, (2) their exact format and field layout, and (3) the sequence in which they must be transmitted. Any two systems that implement the same protocol specification can communicate with each other, regardless of differences in implementation language, hardware, or operating system.

For example: if a web browser and a web server both implement HTTP, they can exchange requests and responses correctly — neither needs any knowledge of how the other is internally built. The protocol specification is the shared contract.

Encapsulation / Piggybacking ↗ ~21:57
As data moves down the protocol stack from sender to receiver, each layer wraps the data it receives from the layer above with its own header, turning it into a payload for the layer below. This process — application data → Transport header added → becomes transport segment → Network header added → becomes packet → and so on — is called encapsulation.

At the receiving end, the process runs in reverse: each layer strips its own header and passes the remaining payload upward. A critical property of this design is that only the layer which added a header can correctly interpret it — all other layers treat it as opaque data they are not permitted to inspect.

OSI Model — 7 Layers ↗ ~08:01

The OSI (Open Systems Interconnection) model defines seven layers. This course uses the TCP/IP 5-layer model, but OSI provides finer granularity — particularly in its decomposition of the upper three layers, which TCP/IP collapses into a single Application layer:

7 · Applicationnetwork apps
6 · Presentationformatting, encryption
5 · Sessionestablishes & maintains sessions
4 · Transportend-to-end delivery
3 · Networkpacket routing
2 · Data Linkframing, MAC, error check
1 · Physicalraw bits over medium
In TCP/IP, OSI layers 5, 6, and 7 are consolidated into a single Application layer. Additionally, some textbooks treat Physical and Data Link as a single "Link" layer — making TCP/IP a 4-layer model — while others count them separately, yielding a 5-layer model. Both representations are valid; the choice is a matter of how finely one wishes to distinguish the lower-level responsibilities.

Network Devices — Layer and Function ↗ ~28:06

Layer 1 — Repeater / Hub

Operates exclusively at the Physical layer — it amplifies and rebroadcasts electrical signals with no awareness of the data they carry. A hub has no concept of MAC addresses, frames, or routing: it simply regenerates the signal on all ports. Because all ports share the same bandwidth and the same collision domain, an 8-port 10 Mbps hub offers only 10/8 Mbps effective throughput per device. If two devices transmit simultaneously, a collision occurs and both transmissions are lost.

⚠️ Hubs cannot filter traffic. Any frame transmitted by one device is received by every other device on the hub. This combination of wasted bandwidth and collision vulnerability is why hubs have been entirely replaced by switches in modern networks.

Layer 2 — Bridge / Switch ↗ ~29:36

Operates at the Data Link layer. A switch maintains a MAC address table and uses it to forward frames only to the port where the destination device resides — all other ports are unaffected. This eliminates the shared collision problem: each port on a switch is its own isolated collision domain. A network interface card (NIC) in a host also operates at Layer 2.

Important limitation: Switches do not segment broadcast domains. A broadcast frame sent by any device is forwarded to every device on the switch. To isolate broadcast traffic, a Layer 3 device (router) is required.

Layer 3 — Router ↗ ~30:20

Operates at the Network layer, working with IP addresses rather than MAC addresses. Routers are the devices that connect separate networks — a switch cannot forward traffic between networks with different IP subnets, but a router can. When a router receives a broadcast frame, it drops it rather than forwarding it, which means routers isolate both collision domains and broadcast domains simultaneously.

Capability hierarchy: A higher-layer device possesses all the capabilities of every layer below it. A router (L3) can also perform all functions of a switch (L2) and a repeater (L1). The reverse is not true — a switch cannot perform routing.

NIC — Network Interface Card

The hardware component that connects a host to a network. Each NIC is assigned a globally unique MAC address (Media Access Control address) by its manufacturer. This 48-bit hardware address is the identifier used at Layer 2 for hop-to-hop frame delivery within a network segment.

Format: 6 bytes (48 bits) — AA:BB:CC:DD:EE:FF

Collision Domain vs. Broadcast Domain
Collision Domain
A set of devices where simultaneous transmission results in a collision. Layer 2 switches eliminate this problem by giving each port its own dedicated collision domain. Layer 1 hubs do not — all ports share one domain.
Broadcast Domain
A set of devices that all receive any broadcast frame sent within the domain. Only Layer 3 routers split broadcast domains. Layer 2 switches forward broadcasts to all ports and therefore do not provide isolation.
LAN vs. WAN ↗ ~14:53

LAN (Local Area Network): Covers a small geographic area such as a building or campus. The organization owns and manages the infrastructure. Uses broadcast topology — all devices can potentially communicate with all others by default.

WAN (Wide Area Network): Spans large geographic distances. Organizations typically lease WAN connectivity from service providers rather than owning the physical infrastructure. The internet is, at the highest level of abstraction, a massive WAN.
