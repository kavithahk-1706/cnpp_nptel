Lecture 54 Medium Priority

Wireless LANs

Overview of WLAN concepts at Layer 1 and Layer 2 — IEEE 802.11 architecture, wireless-specific MAC challenges (hidden/exposed terminal), and the CSMA/CA access mechanism.

Why Wireless LAN?

The proliferation of mobile devices (laptops, phones, PDAs) created an obvious demand for wireless local area networking. Bandwidth availability on wireless data networks has grown steadily, and with 4G/5G deployments, wireless infrastructure is becoming a de facto standard — eliminating the need for physical cable infrastructure entirely. Standardisation is handled by the IEEE 802 committee.

WLAN is a large topic in its own right — this lecture is a focused overview of the Layer 2 (MAC) and Layer 1 (PHY) considerations specific to 802.11. Everything from Layer 3 upward is unaffected.

Categories of Wireless Networks
Infrastructure / Base Station
All communication is routed through an Access Point (AP). The AP connects to a wired backbone, giving controlled, managed connectivity to the wider internet.
Nodes can be fixed or mobile within AP range.
Ad Hoc / MANET
No central AP. Nodes communicate directly with each other.
MANETs (Mobile Ad Hoc Networks) extend this to fully mobile nodes — e.g. vehicular networks (V2V).
802.11 Architecture

Basic Service Set (BSS) and Extended Service Set (ESS)

A BSS is the fundamental building block — a group of stations within mutual radio range. It can be either an ad hoc network (BSS without AP) or an infrastructure network (BSS with an AP).

An ESS connects multiple BSSs via APs, which are wired together through a Distribution System (DS) — typically an Ethernet backbone — that connects further to a server or gateway.

Station Transition Types

No-Transition → Stationary or moving only within the same BSS
BSS-Transition → Moving from one BSS to another within the same ESS
ESS-Transition → Moving from one ESS to a different ESS
Important: IEEE 802.11 does not guarantee continuous communication during a transition. The protocol does not handle seamless handover natively.

MAC Sublayer Challenges

Why Wired MAC Doesn't Work Here

In 802.3 Ethernet, "seizing the channel" is well-defined — CSMA/CD works because every station can hear every other station. In wireless, this assumption breaks down because stations have limited, heterogeneous radio ranges. Two unique problems arise.

Hidden Terminal Problem

Scenario: C is transmitting to B. A wants to transmit to B. A cannot hear C (outside C's range), so A senses the channel as idle and begins transmitting — causing a collision at B that A never detects. Simple CSMA fails because carrier sensing doesn't see what it can't hear.

Exposed Station Problem

Scenario: A is transmitting to some node. B wants to transmit to C. B hears A's transmission and falsely assumes the channel is busy — so B holds back even though a B→C transmission would succeed without interference. B is unnecessarily blocked.
Hidden Terminal
Transmitter is invisible to a third node. Station transmits when it shouldn't — causes collision at receiver.
Under-sensing: senses channel as idle when it isn't (from receiver's perspective).
Exposed Station
Transmitter withholds when it shouldn't. Station holds back even though its transmission would not interfere.
Over-sensing: senses channel as busy when that activity is irrelevant to the intended transmission.
Distributed Coordination Function (DCF)

CSMA/CA — Why Not CSMA/CD?

CSMA/CD requires simultaneous send and receive to detect collisions. Wireless channels are noisy, signals fade over distance, and mobile devices lack the resources for full-duplex collision detection. CSMA/CD is simply not feasible for WLANs — so 802.11 uses Collision Avoidance instead of collision detection.

DCF is mandatory in all 802.11 implementations. It supports two carrier sensing methods:

Virtual Carrier Sensing (MACA)
Uses RTS/CTS handshake and the Network Allocation Vector (NAV) to inform all nearby stations how long the channel will be occupied — without physically monitoring the RF medium.
1-Persistent Physical Carrier Sensing
Station physically senses the channel before transmitting. If idle, transmit. If busy, defer until idle. On collision, apply binary exponential backoff.

MACA: RTS / CTS Handshake

Before sending data, the sender broadcasts an RTS (Request to Send) frame — containing the intended transmission duration. The receiver responds with a CTS (Clear to Send). Any station that hears either frame sets its NAV (Network Allocation Vector) — a timer telling it how long to hold off before checking the channel again.

Station hears RTS → Sets a long NAV waits full RTS duration before sensing
Station hears CTS → Sets a shorter NAV D in B's range: defers for CTS duration
NAV = virtual silence. Stations don't physically sense the medium — they read the duration field in RTS/CTS frames and set a countdown. Only after NAV expires do they check for channel idleness. This is why it's called virtual carrier sensing.

Full CSMA/CA Flow (with RTS/CTS)

1→Set backoff = 0. Apply persistence strategy if channel is busy.
2→Wait for DIFS (Distributed InterFrame Space).
3→Send RTS to destination. Start timer.
4a→If CTS not received before timeout → increment backoff, wait backoff period, retry from step 2.
4b→If CTS received → wait SIFS (Short InterFrame Space).
5→Send data frame. Start timer.
6a→If ACK not received before timeout → backoff and retry.
6b→If ACK received → success.

1-Persistent Physical Carrier Sensing

In this mode, the station physically listens to the RF medium rather than relying on NAV. It senses before transmitting, and does not sense while transmitting (no full-duplex detection). If busy, it waits until idle, then transmits. On collision: binary exponential backoff.

Collisions can still occur in CSMA/CA — the wireless environment is inherently noisy and lossy. The mechanism reduces probability of collision, not to zero. If RTS or ACK is not received within the timer period, the station retries after backoff.
Point Coordination Function (PCF)

Centralised, Collision-Free Access

PCF uses the AP (base station) to poll individual stations to ask if they have frames to send. Because only the polled station transmits at any time, collisions are eliminated entirely.

No collisions AP controls transmission order Optional — not mandatory More complex to implement

Additional capabilities: the AP sends periodic beacon frames for timing synchronisation, and can instruct stations to enter sleep mode to conserve battery — holding their frames until they wake.

DCF and PCF Co-Existence

InterFrame Spacing (IFS) Types

DCF and PCF coexist on the same channel by using different interframe spacings — mandatory waiting gaps between transmissions. Shorter gap = higher priority.

Acronym Full Name When Used Priority
SIFS Short InterFrame Space Between frames in an ongoing dialog: RTS → CTS → Data → ACK Highest
PIFS PCF InterFrame Space When no SIFS response; AP can issue beacon or poll High
DIFS DCF InterFrame Space When no PIFS; any station may contend for channel Normal
EIFS Extended InterFrame Space Reporting bad or unknown frames Lowest
Priority order: SIFS > PIFS > DIFS > EIFS. Because PCF uses a shorter wait than DCF, the AP always gets channel access before any individual station attempting contention-based access.
Concept Summary

Key Takeaways

BSS / ESS
BSS = one AP's coverage zone. ESS = multiple BSSs connected via a wired distribution system. Stations can be stationary (No-Transition), move within an ESS (BSS-Transition), or cross ESS boundaries (ESS-Transition). 802.11 does not guarantee seamless handover.
Hidden / Exposed Terminal
Hidden: A can't hear C → A transmits to B while C is transmitting to B → collision. Exposed: B hears A transmitting → unnecessarily holds back its own unrelated transmission to C. Both are uniquely wireless problems; CSMA/CD cannot solve them.
CSMA/CA + RTS/CTS + NAV
RTS announces intent to transmit and duration. CTS confirms and echoes duration. All overhearing stations set their NAV countdown accordingly and stay silent. No physical collision detection — avoidance through handshake and virtual reservation instead.
DCF (Mandatory)
Distributed contention-based access. Uses CSMA/CA with RTS/CTS/NAV (virtual sensing) or 1-persistent physical carrier sensing. Binary exponential backoff on collision. Required on every 802.11 device.
PCF (Optional)
Centralised poll-based access via AP. Collision-free. AP sends beacons periodically; can command stations to sleep and buffers frames for them. More complex — optional in practice.
