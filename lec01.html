Introduction & Network Architecture Overview

Lecture Overview

This lecture provides a broad, conceptual overview of network architecture — the foundational context for everything that follows in the course. It is primarily concerned with establishing why networks are structured in layers, and what each layer is responsible for. The primary focus is the TCP/IP protocol stack: understanding what each layer does, why the separation of concerns exists, and how real-world networking devices map onto this model.

Protocols — Definition
A protocol is an agreed-upon set of rules that governs communication between two or more parties. Networks are built from multiple layers of protocols stacked on top of one another — each layer handling a specific, well-defined aspect of the communication process — hence the term protocol stack.
The TCP/IP Protocol Stack ↗ ~23:10

Every time data moves between machines, it passes through each of these layers in sequence. Each layer has a specific, well-defined responsibility and communicates only with the layers directly above and below it:

ApplicationHTTP, FTP, SMTP
TransportTCP, UDP, RTP
NetworkIPv4, IPv6, MPLS
Data LinkEthernet, Wi-Fi, Bluetooth
Physicalwires, signals, radio
Consider sending a letter: you write the message (Application), seal it in an envelope (Transport), write the address (Network), hand it to the postal service (Data Link), and a vehicle physically transports it (Physical). Crucially, each layer only interacts with the layer directly above or below it — the postal service does not need to know what the letter says to deliver it correctly.

Why Layers Exist — The Problem They Solve

Physical layer: The foundation — responsible only for transmitting raw signals. Two machines connected by a wire can exchange data at this level. Handles the conversion between digital data and the analog or electrical signals that travel over the medium. ↗ ~06:49

Data Link layer (L2 Switch): When more than two machines share the same network segment, unregulated transmission causes collisions — multiple devices transmitting simultaneously, corrupting each other's data. The Data Link layer, implemented by switches, prevents this by maintaining a table of MAC addresses and forwarding frames only to the correct port. This creates separate collision domains per port. However, all devices on the same switch still share a single broadcast domain. ↗ ~08:27

Network layer (Router): Switches operate only within a single network — they cannot forward traffic across different networks. Routers operate at the Network layer, working with IP addresses rather than MAC addresses, and are responsible for finding the best path between source and destination across multiple interconnected networks. Routers also break up broadcast domains, since they do not forward broadcast frames between networks. ↗ ~14:10

Transport layer: A single machine runs multiple processes simultaneously (a browser, a media player, a messaging app). The Network layer can deliver a packet to the correct machine, but it cannot determine which process should receive it. The Transport layer solves this using port numbers to multiplex and demultiplex traffic between processes. It also provides end-to-end error control, flow control, and traffic management. ↗ ~20:50

Key Vocabulary
Collision domain — A group of devices where simultaneous transmission causes a collision. Layer 2 switches split collision domains by providing a dedicated domain per port.

Broadcast domain — A group of devices that all receive each other's broadcast frames. Only Layer 3 routers split broadcast domains — switches do not.

Intermediate devices — Process packets only up to the layer at which they operate. A hub processes only at Layer 1 (physical signals). A switch processes up to Layer 2 (frames, MAC addresses). A router processes up to Layer 3 (packets, IP addresses). Higher-layer devices have the full capability of all layers below them.

Course Approach

This course follows a top-down approach — beginning at the Application layer and working down toward the Physical layer. Kurose & Ross (the primary textbook) uses the same ordering. Tanenbaum's Computer Networks takes the opposite, bottom-up approach. Both are valid framings; the top-down approach tends to be more intuitive because it starts with protocols you already interact with (HTTP, DNS) before descending into the lower-level mechanics.
